- Yes, create a new staging database, such as `tekg2_1` or `tekg2_0413`, before any import.
- Reason: you now have a new main graph import bundle, a new disease classification layer, and a new tree lineage layer. Importing them into the existing `tekg2` directly would make rollback and validation harder.

  2. Validate node counts, relation counts, disease classification links, and tree lineage links.
  3. Only after validation, switch the application config to the new database if needed.
- This is the safer choice and avoids contaminating the current `tekg2` database.
