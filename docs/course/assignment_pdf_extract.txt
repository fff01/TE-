《生物学数据库及实验》课程大作业
主题二：转座元件（TE）知识图谱与智能问答系统构建
一、 项目背景与目的
转座元件（Transposable Elements, TEs），又称“跳跃基因”，在基因组演化、基因表达调
控以及多种人类疾病（如癌症、神经退行性疾病）中发挥着关键作用。随着相关研究的爆发
式增长，传统的文献检索方式已难以满足科研人员快速获取结构化知识的需求。
本项目要求同学们综合运用生物信息学、自然语言处理（NLP）、图数据库技术以及大
语言模型（LLM），从海量文献中挖掘转座子与生物学功能、疾病、表型性状之间的关联，
构建一个可视化的转座元件知识图谱网站，并实现基于本地知识库的智能问答功能。
二、 核心数据与处理流程
本项目的数据源为公共文献数据库，核心工作流分为以下几个步骤：
1. 文献获取： 利用自动化脚本从 PubMed 批量下载与转座元件相关的文献摘要
（Abstracts）及元数据（发表年份、期刊、作者等）。
2. 实体识别（NER）：从文本中识别出核心实体，包括：转座元件名称（如 Alu, LINE-1,
LTR）、疾病名称、生物学过程/功能、表型性状等。
3. 关系抽取（RE）： 提取实体之间的关联关系（例如：“LINE-1” [促进] “肿瘤发生”）。

三、 系统功能模块与开发指南
为保证项目顺利实施，网站及后台需包含以下核心模块。以下提供了具体的实现思路与
技术栈建议：
1. 文献挖掘与信息抽取模块（核心难点）
 数据获取： 建议使用 Python 的 Biopython 库（Entrez 模块）或直接调用 NCBI
E-utilities API，通过关键词（如 "Transposable Elements", "Retrotransposon",
"LINE-1"）批量抓取文献摘要。
 传统文本挖掘： 针对具有明显命名规律的转座子实体，可构建本地词典，结合正则
表达式（Regular Expressions）进行精确匹配提取。
 深度学习文本挖掘：
o 建议使用基于生物医学语料预训练的模型（如 BioBERT 或 SciBERT）进行
命名实体识别（NER）。
o 可借助 spaCy 的生物医学插件（如 scispaCy）提取疾病和表型实体。
o 对于复杂的关系抽取，鼓励同学们尝试编写 Prompt，调用大模型 API（如
DeepSeek, 智谱 GLM 等）对复杂长句进行零样本（Zero-shot）信息抽取，
输出 JSON 格式的三元组数据（实体 1 - 关系 - 实体 2）。
2. 图数据库搭建模块
 功能要求： 将抽取出的三元组数据结构化并持久化存储，支持高效的图关联查询。
 实现建议： 强烈建议使用 Neo4j 图数据库。
o 节点（Nodes）设计：设置不同的节点标签（Labels），如 TE, Disease, Function,
Trait, Paper。
o 边（Edges）设计： 定义关系类型（Relationships），如 ASSOCIATED_WITH,
REGULATES, MENTIONED_IN，并将文献的 PMID 作为边的属性（Properties）
存储，以便溯源。
o 同学们需要学习编写 Cypher 查询语言，通过 Python 的 neo4j 驱动包将
清洗好的数据批量导入数据库。
3. 知识图谱可视化展示模块
 功能要求： 提供一个直观的 Web 界面。用户搜索某个转座子或疾病时，网页能以
网络图的形式展示其关联的所有知识节点。
 实现建议：
o 前端网络图渲染推荐使用 Cytoscape.js、Vis.js 或 ECharts（关系图组件）。
o 实现交互功能：支持节点的拖拽、缩放；点击某个节点可展开其相邻节点；
点击连线可显示得出该结论的原始文献摘要及链接。
4. 基于大模型的智能问答系统（重点创新模块）
 功能要求： 在网站中集成一个类似 ChatGPT 的对话框。用户可以用自然语言提问
（如：“Alu 元件在阿尔茨海默症中起什么作用？”），系统需结合本地图数据库中的
知识给出准确回答，并附带参考文献。
 实现建议（RAG 架构）：
o Text2Cypher： 接收用户的自然语言问题，通过调用外部大模型 API，将其
转化为 Neo4j 的 Cypher 查询语句。
o 知识检索： 在本地 Neo4j 数据库中执行该 Cypher 语句，检索出相关的子
图路径（如 Alu -> 关联 -> 阿尔茨海默症，出处：PMID 123456）。
o 增强生成： 将检索到的结构化知识作为背景上下文（Context），与用户的
原始问题拼接成新的 Prompt，再次输入给大模型，让其生成一段通顺、严
谨的学术解答。
o 建议了解并使用 LangChain 框架来串联上述流程。
